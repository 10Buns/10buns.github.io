Git Commmit mesagge 和 Change log 使用

Posted on 2019-11-28 | In Git | 次

一、规范的 Commit message 作用

  1. 方便快速浏览
    格式化 message , 每行头部信息即可知道 commit 意义

  1. 快速查找
    查找本次修复的提交

  2. 输出 Change log

二、Commit message 格式

采用Google Angular 规范

header 必须的， body 和 footer 可缺省

2.1 Header

首行 header 包含: type(必需)、scope(可选)、subject(必需)

  1. type 用于说明 commit 类别，限定如下类别:

    • feat:新功能（feature）
    • fix:修补bug
    • docs:文档（documentation）
    • style:格式（不影响代码运行的变动,空格,格式化,等等）
    • refactor:重构（即不是新增功能，也不是修改bug的代码变动）
    • perf:性能 (提高代码性能的改变)
    • test:增加测试或者修改测试
    • build:影响构建系统或外部依赖项的更改(maven,gradle,npm 等等)
    • ci:对CI配置文件和脚本的更改
    • chore:对非 src 和 test 目录的修改

    • revert:Revert a commit

      输出 change log 时， 肯定包含 feat、fix

  2. scope 说明 commit 影响的范围

  3. subject commit 的简述

    • 动词、第一人称
    • 首字母小写
    • 结尾无句号

2.2 Body

对 commit 的详细描述，可拆分多行

2.3 Footer

  1. 不兼容的变更
    如果当前变更与之前不兼容，以BREAKING CHANGE:开头，紧跟描述
  2. 关闭 issue
    Closes #123, #456, #789

2.4 Revert

如果当前 commit 是为了回退历史的 commit，则以revert:开头，紧跟待撤销的 commit 的 header
