手写一个简易的 spring 框架方便理解

Posted on 2020-10-11 | In 后端 | 阅读数

在 Java 开发中， Spring 是用的最多的一个框架，不管是为了加强技术深度还是为了面试，总会多少阅读点 Spring 的源码。但随着迭代，Spring 代码越来越庞杂，不利于理解其核心思想。

之前就想通过手写一个简易版的 demo，进行梳理下Spring的工作流程，抽丝剥茧的方式方便理解。

节前看 Java Guide 的博客，实现了比较完整的 spring boot，但为了避免抄袭(才不是能力不行)，只实现spring简单的接口访问，包括如下:

  1. 自动装配
  2. 包扫描
  3. request 注解
  4. controller 注解
  5. service 注解

最小化演示 Spring 框架如何运行起来，web容器基于 jetty，简化操作。
